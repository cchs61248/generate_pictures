# 前端啟動說明

本目錄為 **Vite + React + TypeScript** 介面，透過環境變數連到本機 FastAPI（預設 `http://127.0.0.1:8000`）。

## 環境需求

- **Node.js**：建議 LTS 版本（例如 20.x），需內建 **npm**。
- 完整使用本專案時，請**另開終端機**先啟動後端 API，再啟動前端（見下方「與後端搭配」）。

## 第一次使用：安裝依賴

在專案根目錄下進入 `frontend`，安裝 npm 套件：

```powershell
cd frontend
npm install
```

若出現 `Could not read package.json`（ENOENT），代表目前目錄不是 `frontend`，或 `package.json` 遺失；請確認路徑為 `…\gnerate_pictures\frontend` 且該資料夾內有 `package.json`。

## （選用）API 位址設定

開發時若後端**不是** `http://127.0.0.1:8000`，可設定 Vite 環境變數：

1. 將 `.env.example` 複製為 `.env`（與 `package.json` 同層）。
2. 編輯 `.env` 中的 `VITE_API_BASE_URL`，指向你的 FastAPI 基底網址（**勿**在尾端多餘斜線）。

未設定時，程式會使用預設：`http://127.0.0.1:8000`。

## 啟動開發伺服器

```powershell
cd frontend
npm run dev
```

成功後終端機會顯示本機網址，預設為：

- **http://localhost:5173/**

瀏覽器開啟該網址即可使用介面。修改原始碼時會熱更新（HMR）。

## 與後端搭配

1. 先啟動 FastAPI（專案根目錄，範例）：

   ```powershell
   .\.venv\Scripts\python.exe -m uvicorn api.server:app --host 127.0.0.1 --port 8000
   ```

2. 再執行 `npm run dev` 開啟前端。

介面標題列會顯示目前使用的 API 位址；若後端未開，上傳或執行流程會連線失敗。

## 其他常用指令

| 指令 | 說明 |
|------|------|
| `npm run dev` | 開發模式（Vite） |
| `npm run build` | 型別檢查（`tsc`）並建置到 `frontend/dist` |
| `npm run preview` | 在本機預覽建置結果（需先 `npm run build`） |
| `npm run lint` | 執行 ESLint |

## 疑難排解

- **`npm run dev` 找不到 `package.json`**：請在 `frontend` 目錄執行，並確認已執行過 `npm install`。
- **前端能開但無法上傳／執行**：檢查後端是否已在對應埠運行、防火牆是否阻擋，以及 `.env` 的 `VITE_API_BASE_URL` 是否正確。
- **埠號被占用**：關閉占用 5173 的程式，或在 `vite.config.ts` 設定 `server.port` 為其他埠。
